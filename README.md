# SofLIA - Engine Render Worker

Local Remotion render worker for SofLIA - Engine.

## Development commands

```powershell
npm install
npm run build
node dist/cli.js ui
node dist/cli.js doctor
node dist/cli.js start
```

Open the local URL printed by `ui`, paste the temporary `SLIA-000000` code generated by SofLIA - Engine, and the worker stores its limited token locally. `start` keeps the worker online and automatically claims eligible desktop render jobs.

`link --api-url ... --code ...`, `render --job-id ...`, and `configure --api-url ... --token ...` remain available as development fallbacks.

The CLI never uses `SUPABASE_SERVICE_ROLE_KEY` and never talks to Supabase directly. It only calls the SofLIA - Engine worker HTTP API with a limited worker token.

## Electron app

Run the desktop app in development:

```powershell
npm run electron:dev
```

This command builds the Electron main process, starts the Vite renderer, and opens the app.

Build the Windows installer:

```powershell
npm run package:win
```

The installer is generated under `release/`. Upload the generated `.exe` to the chosen public download location and set this in SofLIA - Engine web:

```env
NEXT_PUBLIC_SOFLIA_WORKER_DOWNLOAD_URL=https://.../SofLIA-Engine-Render-Worker-Setup.exe
```

The Electron app keeps the same backend contract as the CLI. In local development use `http://localhost:3000` as the API URL. In production use the Netlify/SofLIA - Engine site URL.

The UI includes a `Mantener en segundo plano al cerrar` setting. When it is enabled, closing the window hides the app and keeps the worker alive. Use `Salir de la app` from the UI or `Salir` from the tray menu to stop the worker and close the program completely.
